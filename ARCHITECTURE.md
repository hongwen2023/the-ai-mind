# The AI Mind Architecture

## Purpose

This document explains **why the project is structured the way it is**,
not the current implementation status.

## Why Four Books?

Book I --- Discovering Intelligence

- Build mental models.
- Understand representation, learning and neural networks.
- Minimize prerequisites.

Book II --- Learning to Learn

- Mathematics, probability, optimization and classical machine learning.

Book III --- Learning to Build

- Implement modern AI systems from first principles.
- micrograd → makemore → Transformer → GPT → LLM Engineering.

Book IV --- Learning to Create

- Research methods, AI systems, agents and capstone projects.

## Why Book I Has 30 Chapters?

Thirty chapters provide a balanced progression:

- Part I (1--10): Thinking.
- Part II (11--20): Mathematical foundations of learning.
- Part III (21--30): Representation learning to modern AI.

The chapter count is frozen for Book I v1.x to preserve references.

## Why There Is an Unnumbered Prelude

The Prelude is the shared entrance to all four books. It builds the
learner's first mental map and explains why the curriculum moves from
intuition to mathematics, implementation, engineering, and research.

It is not Chapter 0 and does not alter Book I's frozen count. Historical
Journey 00 material may inform it, but the canonical Prelude is rewritten
under the current Author / Editor workflow.

## Architectural Principles

-   GitHub repository is the single source of truth.
-   One chapter equals one Markdown document.
-   Stable architecture, evolving content.
-   Documentation and code evolve together.
