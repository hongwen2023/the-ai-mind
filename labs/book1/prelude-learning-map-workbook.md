# Prelude Learning Map Workbook

**Purpose:** Turn the Prelude's map into the learner's own working model.

**Estimated time:** 45-60 minutes

**Required tools:** Paper or a blank digital canvas

This is not a reading quiz. Complete it without copying the Prelude's diagrams.

## 1. Draw the Map From Memory

Draw a path from reality and data to an AI system that can act and learn.

Your map must include at least these functions, but you may rename them:

- representation;
- computation or modeling;
- objective or feedback;
- parameter update;
- evaluation;
- action.

For every arrow, write one sentence explaining why the dependency exists.

## 2. Open a Black Box

Choose one:

- `loss.backward()`;
- Transformer attention;
- RAG;
- an AI research agent.

Complete the table:

| Question | Your answer |
|---|---|
| What problem does it solve? | |
| What representation does it consume? | |
| What computation does it perform? | |
| What feedback or objective shapes it? | |
| What can fail? | |
| Which earlier ideas must I understand? | |

## 3. Compare Two Rule Sources

Give one task suited to explicit rules and one task suited to learning from
data. For each, explain:

1. where the rules come from;
2. how errors are detected;
3. who is responsible when the result is wrong;
4. what evidence would make you redesign the system.

## 4. Build the Mathematics Bridge

Complete each sentence in your own words:

- Linear algebra helps AI represent ...
- Calculus turns error into ...
- Probability lets a model express ...
- Optimization searches for ...

Then draw one system in which all four appear together.

## 5. AI x Finance Mapping

Choose one investment research question. Examples:

- Will a company's gross margin structurally improve?
- Is an AI infrastructure supplier's growth durable?
- Has the market mispriced a regulatory risk?

Map the research process:

| Learning-system element | Investment equivalent | Concrete choice |
|---|---|---|
| Data | Evidence set | |
| Representation | Company / industry representation | |
| Objective | What a useful forecast should optimize | |
| Prediction | Testable claim | |
| Error | Evidence that would prove the claim wrong | |
| Update | What changes after new information | |
| Portfolio decision | Action under risk constraints | |

## 6. Teach Back

Record or write three explanations of “why learning AI needs a map”:

1. 60 seconds for a 12-year-old;
2. 90 seconds for a software engineer;
3. 90 seconds for a portfolio manager.

Do not reuse the same analogy three times. Adapt the explanation to what each
listener already knows.

## 7. Exit Check

Mark each statement only when you can support it with evidence:

- [ ] I can redraw the AI learning loop without looking.
- [ ] I can distinguish explicit rules from learned rules.
- [ ] I can explain why `micrograd` comes before GPT implementation.
- [ ] I can place linear algebra, calculus, probability, and optimization on one map.
- [ ] I can name one place where the map oversimplifies reality.
- [ ] I can transfer the map to an investment research process.

Any unchecked item becomes a question to revisit in Chapter 1.
