## Spiral Matrix

The matrix tracks when a concept returns with a genuinely new purpose. A later
appearance should deepen or transfer the concept, not repeat its first
explanation.

| Concept | First encounter | Second layer | Later return |
|---|---|---|---|
| Understanding | Prelude: maps and relationships | Ch01: Explain / Predict / Reconstruct / Transfer | Ch09 review and every Learning Package Audit |
| Rules and relationships | Ch01: rule compression | Ch02: iteration, interaction, and state | Ch13-14: layers and nonlinearity; Ch21-30: modern AI behavior |
| Abstraction | Prelude: map hides detail | Ch03: abstraction as a reasoning tool | Ch12-14: matrix and layer interfaces; Ch21-30: system composition |
| Representation | Prelude: world to computable form | Ch04: representation as a core problem | Ch11-12: vectors and matrices; Ch21-30: learned representations |
| Learning and feedback | Prelude: `Learning = Computation + Feedback` | Ch06: why machines can learn | Ch15-19: gradient, loss, optimization, chain rule, backpropagation |

| Complexity and emergence | Ch02: local rules to global behavior | Ch13-14: simple transformations compose | Ch21-30: emergence and modern architectures |
| AI x Finance | Prelude: research as a learning loop | Ch01: driver tree; Ch02: market feedback | Revisited when each mechanism creates a genuine finance transfer |
