# Editorial Style Guide

## Language

- 中文为主，首次出现的重要术语标注英文。
- 避免不必要的术语堆叠。
- 公式之前先建立直觉。
- 不使用碎片化短句作为正式正文风格。

## Chapter structure

1. Learning Objectives
2. Opening Question / Story
3. Feynman Explanation
4. Intuition
5. First Principles
6. Mathematics
7. Visualization
8. Coding Lab
9. Engineering Perspective
10. AI × Finance
11. Failure Modes / Common Mistakes
12. Research Note
13. Exercises
14. Teach Back
15. Master Insight
16. Bridge to Next Chapter

## Code

- Python 3.11+
- Type hints for reusable code
- NumPy before PyTorch when pedagogically useful
- Explain tensor shapes explicitly
- Every lab must include expected observations

## Versioning

- `draft`: incomplete or under active development
- `review`: complete and awaiting technical/editorial review
- `v1.0`: internally consistent learning edition
- `v2.0`: Discovery Edition
